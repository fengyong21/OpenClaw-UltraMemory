# 🦞 OpenClaw-UltraMemory V4

基于**热度钉扎滑动窗**的记忆系统，融合 **Hermes Agent** 的自动技能生长机制。

> V1 SimHash 检索 → V2 Parent_ID 链路 → V3 热度滑动窗 → **V4 父子链上下文 + 自动技能生长**

---

## 铁律：不可篡改 + 行为可追溯

本项目与 Hermes 的根本区别在于**信任模型**：

| | Hermes | OpenClaw-UltraMemory |
|--|--------|---------------------|
| 信任模型 | 隐式信任框架 | 零信任，代码自证 |
| 原始数据 | 框架可读写 | 追加只读，框架无权修改 |
| 行为审计 | 黑箱，依赖官方 | 全透明，明文脚本可逐行审查 |
| 追责能力 | 出问题找官方 | 行为自证，完整回溯 |

**两条铁律，永远不可打破：**

1. **原始数据不可篡改**：`raw/` 目录下的 `.md` 文件只追加，不修改、不覆盖、不删除。任何脚本都无权触碰已写入内容。
2. **行为全程可审计**：所有操作通过明文 Python 脚本执行，用户可随时审查代码验证行为。迁移时只需迁移 `raw/` + `hot_window.db`，新环境可完整重建。

## 父子链：解决数据膨胀后逻辑断裂

### 核心字段

| 字段 | 说明 |
|------|------|
| `id` | 主键自增 |
| `parent_id` | 父节点 ID，None=根节点 |
| `raw_link` | 指向 raw/ 原文路径 |
| `heat` | 热度值，强化递增，衰减递减 |
| `timestamp` | 时间戳，辅助排序 |
| `summary` | 一句话摘要 |

### 写入时自动关联

```
对话A → parent_id=None（根节点）
对话B → parent_id=A.id
对话C → parent_id=B.id
对话D → parent_id=C.id
```

### 检索时链路回溯

```python
trace_chain(record_id=5, depth=5)
# 返回：
# [A(id=2, parent_id=None)]   ← 根
# [B(id=3, parent_id=2)]
# [C(id=4, parent_id=3)]
# [D(id=5, parent_id=4)]       ← 当前
```

## 热度钉扎滑动窗

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `WINDOW_SIZE` | 1000 | 滑动窗口最大条数 |
| `PROTECT_SECS` | 604800 | 7天保护期，新记录不淘汰 |
| `DECAY_RATE` | 0.99 | 每次写入，旧记录热度×0.99 |
| `HEAT_CAP` | 200 | 热度上限，防老霸主垄断 |
| `TOP_K` | 10 | 检索返回条数 |

## 迁移说明

### 从 V2/V3 升级

运行 `install.sh`，hot_window.db 保持兼容。旧记录 parent_id=None（根节点），不影响链路回溯。

### 一键迁移

```bash
cd claw-memory
./export.sh                    # 导出 raw/ + hot_window.db
./import.sh /path/to/export   # 新机器导入
./verify.sh                   # 验证数据完整性
```
