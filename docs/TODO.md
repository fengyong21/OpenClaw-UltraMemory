# 待优化项

> 更新：2026-04-22

---

## 1. 自动化触发机制

**问题**：claw-memory 有存储能力，但无自动触发机制，依赖手动调用。

**现状**：
- 所有类 OpenClaw 记忆系统都存在此问题
- 根因在框架层/系统层，需要底层支持
- 当前无解，只能手动

**解决方向**：
- 等待框架支持 hook
- 或写 wrapper 监听对话事件（门槛高）

---

## 2. 多平台数据迁移

**问题**：各平台（Trae、CodeBuddy CN 等）数据格式不同，迁移时无法读取历史数据。

**解决方向**：
- L2 只迁移元数据（timestamp、session_id、parent_id、platform）
- L1 raw_link 指向原平台路径（或复制一份）
- simhash 按需计算，不全量算

**待做**：
- [ ] 调研各平台数据格式
- [ ] 设计迁移适配器

---

## 3. skill 质量优化

**问题**：当前草稿版 auto_skill 生成的技能质量一般（文本拼接）。

**优化方向**：
- [ ] 质量打分（长度/代码块/关键词密度）
- [ ] 锚点验证（多轮会话才生成）
- [ ] 草稿模式（用户确认后才激活）
- [ ] SimHash 去重（替代简单文本相似度）
